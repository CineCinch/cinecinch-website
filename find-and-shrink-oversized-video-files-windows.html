How to Find and Shrink Oversized Video Files on Windows Without Losing Quality

If you work with screen recordings, phone clips, or exported projects, your drives can fill up fast. Some videos are huge for a good reason, but others are quietly wasting space — using far more megabytes per second than they need to. In this guide, we’ll look at how to find the true space hogs on Windows and shrink them without visible quality loss.

Why some videos are much bigger than they need to be

Video file size is mostly driven by three things:

  • Resolution (e.g., 1080p vs 4K)
  • Duration (how long the video is)
  • Bitrate (how many bits per second are used to encode it)

Two files can have the same resolution and duration but wildly different sizes. That usually means one of them is inefficient — it’s using more megabytes per second than it needs to for the actual content on screen.

The trick is not just to find “big files”, but to find the ones that are unreasonably big for their length. That’s where looking at MB per second becomes much more useful than just sorting by size.

Common ways to shrink video files on Windows

There are three main approaches people use to reduce video file size on Windows:

1. HandBrake (powerful, but manual)

HandBrake is a popular open-source transcoder. You can load a file, pick a preset, and export a smaller version. It’s powerful and free, but it has a lot of knobs: codecs, bitrates, CRF values, filters, and more.

HandBrake is great if you’re comfortable tweaking encoding settings and working file-by-file. It’s less ideal when you just want to quickly clean up a messy folder full of oversized videos.

2. ffmpeg (flexible, but very technical)

ffmpeg is the command-line engine behind many video tools. You can use it directly to compress videos, for example: 

ffmpeg -i input.mp4 -c:v libx265 -crf 24 -c:a copy output.mp4

This tells ffmpeg to re-encode the video using H.265 (HEVC) at a given quality level while copying the audio. It’s extremely flexible, but you have to be comfortable with the command line and know which options to use.

3. Online video compressors (convenient, but slow and risky)

There are many browser-based tools that let you upload a video and download a smaller version. They’re convenient, but they come with trade-offs:

  • Uploading large files is slow and bandwidth-heavy
  • Privacy concerns (you’re sending your content to a third-party server)
  • File size limits and queues

For large libraries or sensitive content, online tools are usually not a great fit.

Why “sort by size” isn’t enough

On Windows, the usual first step is to open File Explorer and sort a folder by size. That’s a start, but it doesn’t tell you which files are actually inefficient.

A 4 GB, 2-hour movie might be perfectly reasonable. A 4 GB, 5-minute screen recording is probably not. What you really want to know is:

  • How many megabytes per second each video is using
  • Which files are outliers compared to the rest of your library
  • Which ones will give you the biggest space savings if you compress them

That’s where tools that understand MB/sec inefficiency become much more useful than just a file size column.

Manual vs automated workflows: when to use which

If you only have a handful of files and you like full control, HandBrake or ffmpeg are great options. You can tune every parameter and inspect each result.

If you have folders full of mixed content — screen recordings, exports, clips from different devices — an automated workflow that:

  • Finds the most inefficient videos by MB/sec
  • Lets you queue them quickly
  • Runs locally without babysitting

will save you a lot of time and guesswork.

Tips for keeping video sizes under control

  • Use efficient codecs: Prefer H.265/HEVC when your playback devices support it.
  • Match resolution to content: Don’t store low-detail screen recordings at 4K if 1080p is enough.
  • Watch MB/sec, not just GB: A short, huge file is often a better compression candidate than a long movie.
  • Batch process: Queue multiple oversized files at once instead of handling them one by one.
